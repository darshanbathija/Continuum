# P0 — Must Fix (Crash / Data Loss / Security / Broken Core Feature)

### [P0] Arbitrary file read via GET /sessions/:id/markdown-document (absolute paths escape session sandbox)
**Files:** `apple/ClawdmeterMac/AgentControl/AgentControlServer.swift:7968` (`standardizedMarkdownDocumentPath`), `3283` (`handleGetMarkdownDocument`), `3292` (call), `3306` (open + read + wire).
**Issue:** For absolute `?path=/...` (or ~ expanded), the function only rejects control bytes + traversal segments in the raw arg; it does NOT require the result (after standardizing) to be under `session.effectiveCwd` or home or any allowlist. It then does `resolvingSymlinksInPath` + `open(resolved, O_RDONLY | O_NOFOLLOW)` + fstat + read + returns content to the caller (paired iOS/Watch or relay client). No re-check of the final resolved path against the session worktree.
**Why P0:** Authenticated paired client (or compromised relay peer) can read any file the daemon uid can read (`~/.ssh/*`, `.env`, source of other projects, Keychain JSONs if permissions allow, etc.) up to 2MB and exfil over the control channel. Violates the "only under session cwd/worktree or explicit agent outputs" model. Contrast with the much stronger artifact handler (which does dual canonical + resolved prefix checks).
**Evidence:** `if expanded.hasPrefix("/") { absolute = expanded }` then only standardize; no `under*` guard like artifact lines 3211-3213; handler proceeds straight to open/read/send. Tests only cover relative traversal + control bytes.
**Fix direction:** Make markdown-document use the same "must resolve under session worktree (or explicit allowlisted review/plan dirs under ~/.gstack or ~/.claude for the review case)" logic as artifact, with the same standardize + resolve + prefix + O_NOFOLLOW + fstat dance. Reject absolute outside that set. Or restrict to relative-only + generated artifacts under known roots.

### [P0] Force-unwrap + nil/empty assumptions in hot paths (multiple instances, including group.next()!, worktreePath!, .first!, URL ctor!)
**Files (examples; cross-cutting agent was enumerating all):** `apple/ClawdmeterMac/SessionsView.swift:1586` (`group.next()!`), `1763` (`worktreePath!`), `2024` (`goal!`); `apple/ClawdmeterShared/Sources/ClawdmeterShared/AgentControl/SDKChatTranscriptMirror.swift:47` (`.first!`); `apple/ClawdmeterShared/Sources/ClawdmeterShared/Cache/FirstPromptCache.swift:101` (prior P0, app support dir .first! — widget/sandbox can be empty); many URL(string: "...")! in constants/drivers; TahoeDemo fallbacks; continuation ! in drivers.
**Issue:** Several in user-action paths (session delete/continue, transcript loading, spawn) and init/widget paths.
**Why P0:** Crash on normal or edge conditions (empty groups after race, missing worktree in conductor/Recent flow, sandboxed widget, network flake on first prompt cache, missing bundled binary path).
**Fix direction:** Guard + graceful degrade (show error state, skip, log + recover) everywhere. Prior audits called these out; some remain.

### [P0] Linux GTK/AppIndicator/Cairo from background actor + @unchecked Sendable on C interop classes
**Files:** `linux/Sources/ClawdmeterLinux/Tray/TrayPollLoop.swift`, `AppIndicatorTray.swift`, `CairoGaugeRenderer.swift` (and LinuxPairingTokenStore etc.); multiple `@unchecked Sendable` on tray/UI classes.
**Issue:** Actor-isolated poll calls non-@MainActor C GTK setIcon / render directly (or via unchecked boxes); GTK is not thread-safe. AppRun LD_LIBRARY_PATH still vulnerable to CWD injection on empty $LD_LIBRARY_PATH (trailing :).
**Why P0:** UB, immediate segfault or memory corruption on Linux desktop start or gauge update; local priv-esc via planted libs when running AppImage from /tmp or cwd.
**Status:** Prior P0s; agents confirmed patterns + unchecked markers still present + umask comments show partial cleanup but not root thread safety.
**Fix direction:** Dispatch all GTK via DispatchQueue.main (or @MainActor isolate the tray/renderer); fix AppRun concat guard; remove or justify every @unchecked on C interop with actual Sendable proof.

### [P0] WCSession delegate contention (last writer wins) between Watch bridges
**Files:** `apple/ClawdmeterShared/Sources/ClawdmeterShared/Sources/WatchTokenBridge.swift`, `WatchPlanBridge.swift` (and WatchUsage?); ClawdmeterWatchApp.swift.
**Issue:** Multiple singletons do `WCSession.default.delegate = self` in init / setup. Whichever initializes last owns delivery of didReceiveApplicationContext / userInfo / activation.
**Why P0:** Intermittent/total loss of token push, usage, plan approvals, session summaries on Watch depending on launch order of bridges. "Waiting for iPhone", stale complications, plan approval on wrong session.
**Fix direction:** Single WCSessionDelegate multiplexer (or one owner that fans out via Combine subjects to the bridges). Never assign delegate more than once.

### [P0] (Potential / confirm with full agent output) Analytics cache dedup + reparse still drops or misattributes provider history; Codex cumulative resets double-count
**Files:** `apple/ClawdmeterShared/Sources/ClawdmeterShared/Analytics/UsageHistoryLoader.swift` (mergePerFileResult, reparse paths), `CodexUsageParser.swift` (delta logic on cumulative dip).
**Issue:** On collision or missing file, reparse path hard-coded or falls back losing non-Claude aggregates. Cumulative dip in Codex JSONL emits full totals as delta.
**Why P0:** User analytics (spend by day/repo/provider) diverge from `ccusage daily` (the documented ground truth) after normal log rotation, session respawn, or cache bump — silent data corruption in the primary "what did I spend" feature.
**Status:** Agents were deep in this slice (60+ calls); prior audits flagged exact cases. Verify if schema bump + parser branching landed.

### [P0] Duplicate / drift in iOS-safe path validator (iOSArtifactsPane still has local copy + IOSMarkdown uses validator)
