## Executive summary

1. **Security:** Paired-client endpoints still trust too much post-auth — arbitrary absolute paths on `GET /sessions/:id/markdown-document`, uneven `repoKey` validation on session spawn (see May audit), and relay/token exposure patterns.
2. **Analytics truth:** Codex cumulative-counter resets inflate totals; cache merge drops history when JSONLs rotate or dedup collisions force Claude-only reparse; OpenCode/Antigravity repo bucketing diverges from ccusage; fabricated Antigravity token counts get priced.
3. **Mac Sessions / Chat (CenterThread cluster):** Highest-impact UX bugs — composer and PR mirror stay bound to the first session after tab switches; read-only→live promotion corrupts offline/pending send state; stale closures and wrong chat-store identity.
4. **Mobile / Watch:** WCSession single-delegate contention; iOS relay/Chat V2 paths bypass outbox and lack HTTP fallback; wrong foreground notification breaks WS reconnect; Watch plan approval can target the wrong session.
5. **Linux:** Package declares macOS-only platform; daemon skeleton + transport stubs; OAuth tokens not persisted; empty bearer accepted.
6. **Idempotency / mobile outbox:** `create-pr`, `merge`, and `review-pr` bypass the outbox replay path; payload-hash mismatch gate rarely fires because handlers omit hashes.
7. **CI / release:** XCTSkip-heavy suites can false-green; five security regression tests still spec'd in `TODOS.md` but not implemented.

# P0 — Must fix first

### P0-1. Codex analytics double-counts on cumulative counter reset
**Location:** `apple/ClawdmeterShared/Sources/ClawdmeterShared/Analytics/CodexUsageParser.swift:84-103`
**Issue:** When any cumulative field dips (session reset mid-rollout), `delta = cumulative` re-emits the **full** session totals instead of a bounded delta.
**Impact:** Analytics totals and cost charts can spike to multiples of ccusage ground truth after Codex respawns or counter resets.
**Fix:** Treat reset as a new baseline (delta = cumulative only for the first event after reset, or zero out and set `previousCumulative = cumulative` without emitting). Add regression fixture from a real rollout with mid-file reset.

### P0-2. Analytics cache silently drops history when source JSONL disappears
**Location:** `apple/ClawdmeterShared/Sources/ClawdmeterShared/Analytics/UsageHistoryLoader.swift:818-851` (`mergePerFileResult`)
**Issue:** On dedup-key collision, reparse reads `result.path` with `try? Data(contentsOf:)`. If the file was deleted/rotated, reparse is skipped and **cached aggregates for that file are never merged** — no error, no fallback.
**Impact:** User sees usage/cl cost drop after log rotation or cache invalidation; diverges from ccusage.
**Fix:** When reparse fails, fall back to cached per-file totals (documented tradeoff) or invalidate that cache entry and force full rewalk. Bump cache schema version on behavior change.

### P0-3. Mac CenterThread binds composer/PR mirror once — wrong session after tab switch
**Location:** `apple/ClawdmeterMac/Workspace/CenterThread.swift:48-80`, `SessionWorkspaceView.swift:106-127`
**Issue:** `CenterThread` uses stable SwiftUI identity across session switches but `@ObservedObject` for `composerStore` and `prMirror` are initialized in `init` only. Comments acknowledge tab-switch perf guard but **do not rebind** on `session.id` change.
**Impact:** Send/refine/plan/PR actions target the **previous** session's composer and PR state — data corruption and wrong tmux/daemon targets in normal multi-session use.
**Fix:** Rebind `composerStore`/`prMirror` in `.onChange(of: session.id)` (or drop stable identity and use `.id(session.id)` on CenterThread). Add UI test: open session A, switch to B, send — assert B's JSONL receives bytes.

### P0-4. Arbitrary host file read via markdown-document endpoint
**Location:** `AgentControlServer.swift:3288-3307`, `7968-7987`
**Issue:** `standardizedMarkdownDocumentPath` allows **absolute** paths outside the session worktree (only blocks traversal/control bytes). Extension guard passes when `pathExtension.isEmpty`. Handler uses `O_NOFOLLOW` but **never re-checks** resolved path is under session cwd/worktree.
**Impact:** Authenticated paired client (stolen token) can read any user-readable file on the Mac (`~/.ssh/id_ed25519`, `.env`, etc.) up to 2MB.
**Fix:** Require path canonicalization under `session.effectiveCwd` (same pattern as `/artifact`). Reject empty extension or whitelist `.md`/`.markdown`. Regression test: `path=/etc/passwd` → 403.

### P0-5. Linux Swift package cannot build on Linux CI/target
**Location:** `linux/Package.swift:14-17`
**Issue:** `platforms: [.macOS(.v14)]` — `swift build` on Ubuntu (per `.github/workflows/linux.yml`) is unsupported/wrong platform declaration. C GTK shims and daemon transport are skeleton/stub.
**Impact:** Advertised Linux AppImage/.deb pipeline cannot produce a working binary; CI may false-pass on macOS cross-compile only.
**Fix:** Declare Linux platform, gate macOS-only deps, wire Hummingbird/SwiftCrossUI per Phase 3 plan, or disable Linux CI until shippable.

### P0-6. watchOS WCSession delegate ownership is undefined
**Location:** `WatchTokenBridge.swift:68-73`, `WatchPlanBridge.swift:42-44`, `ClawdmeterWatchApp.swift`
**Issue:** Both `WatchTokenBridge.shared` and `WatchPlanBridge.shared` set `WCSession.default.delegate = self` in `init`. Whichever singleton initializes **last** wins; the loser never receives `didReceiveApplicationContext` / `didReceiveUserInfo`.
**Impact:** Intermittent loss of plan approvals, session summaries, or token/usage on Watch depending on launch order — "Waiting for iPhone" / stale complications.
**Fix:** Single `WCSessionDelegate` multiplexer that fans out to token + plan bridges; never assign delegate twice.

### P0-7. Mac read-only → live promotion corrupts offline send / pending state
**Location:** `apple/ClawdmeterMac/Workspace/CenterThread.swift` (offline queue, `finishBoundSendWithError`, `pendingFirstSendRecoveries`)
**Issue:** When a synthetic read-only session is promoted to a live registry session, composer offline paths still mutate the **synthetic** `SessionChatStore` while HTTP sends target the **live** session id. `finishBoundSendWithError` queues recovery keyed by live id but leaves pending bubbles on the synthetic store; drain/retry paths POST to the wrong store identity.
**Impact:** User sees sends "stuck" pending, duplicate bubbles, or lost prompts after first-send-from-Recent — normal Sessions workflow.
**Fix:** On promotion, rebind `chatStore` + composer to the live session atomically; key pending/offline queue by final session UUID; disable offline queue for synthetic read-only rows. UI test: open Recent JSONL → send → verify live JSONL receives bytes.

### P0-8. Dedup-collision reparse hard-codes Claude parser — drops non-Claude history
**Location:** `apple/ClawdmeterShared/Sources/ClawdmeterShared/Analytics/UsageHistoryLoader.swift:818-851`
**Issue:** On per-file dedup-key collision, `mergePerFileResult` always re-parses via `ClaudeUsageParser` regardless of whether the path is under `~/.codex/sessions/`, Antigravity DB, or OpenCode logs.
**Impact:** Codex/Antigravity/OpenCode usage vanishes from totals after cache collision or schema bump — silent divergence from ccusage (distinct from P0-2 file-missing path).
**Fix:** Branch on provider from path prefix or cached metadata; use `CodexUsageParser` / `AntigravityUsageParser` / `OpencodeUsageParser` as appropriate. Fixture test per provider.
