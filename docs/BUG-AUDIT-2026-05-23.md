## Executive Summary — Top Themes

1. **Supply-chain attack surface is large and unguarded** — Bundled Node and uv binaries are downloaded into the DMG without checksum/signature verification; the Linux AppImage's `LD_LIBRARY_PATH` setup has a CWD-injection bug; CI uses `@latest` for a third-party GitHub Action; no `permissions:` block on the workflow. Any of these alone can pivot to remote code execution at the user.
2. **`AgentControlServer` boundary enforcement is uneven across paired-client endpoints** — `/sessions` accepts an unvalidated `repoKey` (sibling routes call `isValidRepoKey`); `/design/import-folder` accepts any `baseDir` (so a paired iPhone can ask the Mac to ingest `~/.ssh`); `CodexSubscriptionRelay` logs full prompt bodies as `.public` via OSLog. Auth gating exists but the post-auth surface is too wide.
3. **The OpenCode provider is advertised but not actually wired** — `.opencode` is in `AgentKind.allCases`, `/sessions` will create one, but `/sessions/:id/send` has no opencode branch and falls through to a pane-id guard; SSE message events don't populate snapshots; crash recovery never re-spawns the server. Same theme on Antigravity SDK mode (status says active, runtime still on disk estimates) and Gemini live-activity push (token registered for one variant, not the other).
4. **Plan-approval rollout-swap is broken on Mac and iOS** — Mac's `SessionsModel.chatStore(for:)` and Mac's `DaemonChatStoreRegistry.snapshotStore(for:)` both return the pre-approval cached store; after approving a Codex plan, the UI freezes on the plan and never tails the new execution JSONL.
5. **Linux port is non-shippable end-to-end** — `linux/Package.swift` depends on `ClawdmeterShared` which transitively imports `OSLog`/`Combine`; tests don't compile after an API change; AppImage/deb scripts exit 2; systemd unit flaps because the daemon body is a stub; the tray loop calls GTK from a background actor (segfault risk); Cairo renders empty PNGs; XDG vars are trusted unconditionally.
6. **Release machinery can ship the wrong thing** — Fastlane Match requests the wrong widget bundle ID; release upload picks DMGs by lexicographic sort (`0.9.2 > 0.22.32`); TestFlight lane claims to bundle Watch but the iOS target doesn't embed it.
7. **Concurrency hazards in shared state** — `UsageCloudMirror` is `@unchecked Sendable` with no locking around a read-modify-write of the iCloud KV provider list (lost provider registrations); global `umask()` mutated on Linux fallback writes; PTY fds double-closed across `stop`/`deinit`.
8. **Crash risks in `ClawdmeterShared`** — Four real force-unwraps (TaskGroup, FileManager URLs, dictionary fallback, mtime compare) plus pervasive `try?` silent error swallowing across the persistence layer.
9. **Zombie subprocess pattern across Mac process managers** — Five `process.terminate()` sites missing `waitUntilExit()`; the correct pattern lives in the same files.
10. **Mobile reliability gaps** — Composer drops user text on send failure; broadcast UI labels the wrong provider for selected subsets; iOS Chat broadcast initialized to true even with one provider; Frontier child first-prompt warmup bypassed; cold transcript loading is unbounded; iOS sessions/Live/Watch can go stale until manual refresh.

# P0 — Must fix (crash / data loss / security / supply chain / broken feature)

### [P0] Bundled Node & uv runtimes downloaded without checksum or signature verification
**Files:** `tools/download-bundled-node.sh:61, 75`; `tools/download-bundled-uv.sh:44`
**Issue:** Both scripts `curl -fsSL` the release tarballs and stage them into `apple/ClawdmeterMac/Resources/Vendor/` without fetching `SHASUMS256.txt`, no GPG verify, no per-asset `.sha256`.
**Why it's a bug:** A network-path compromise (malicious local network, BGP/DNS hijack of `nodejs.org`/`github.com/astral-sh/uv`, malicious CI runner, future TLS-MITM) substitutes a trojanized Node or uv. Every downstream user runs the attacker's binary as part of the trusted runtime — bypasses Tailscale, Keychain, HKDF token, and the rest of the threat model.
**Fix:** Fetch the publisher checksums and verify before touch: Node ships signed `SHASUMS256.txt(.sig)`; uv ships per-artifact `.sha256`. Pin Node release GPG keys in repo.

### [P0] Linux AppImage `LD_LIBRARY_PATH` has CWD-injection via empty segment
**File:** `linux/resources/packaging/appimage/AppRun:18`
**Issue:** `export LD_LIBRARY_PATH="$HERE/usr/lib:${LD_LIBRARY_PATH:-}"`. When the inner variable is empty, the result ends with a trailing colon. `ld.so` treats an empty path segment as `.` (CWD).
**Why it's a bug:** Running the AppImage from `/tmp` or any shared directory loads a same-named `libcurl.so` / `libssl.so` planted by an attacker, before the bundled or system version. Local privilege escalation / RCE.
**Fix:** Guard the concatenation: `if [ -n "${LD_LIBRARY_PATH:-}" ]; then ... else export LD_LIBRARY_PATH="$HERE/usr/lib"; fi`.

### [P0] GTK / AppIndicator C calls made from a background actor thread
**Files:** `linux/Sources/ClawdmeterLinux/Tray/TrayPollLoop.swift:64`; `linux/Sources/ClawdmeterLinux/Tray/AppIndicatorTray.swift:12`
**Issue:** `TrayPollLoop` is a cooperative `actor` and calls `tray.setIcon(...)` synchronously; `AppIndicatorTray.setIcon` is a non-isolated plain method that invokes `app_indicator_set_icon_full` directly.
**Why it's a bug:** GTK4/Cairo/GDK are explicitly not thread-safe. The call lands on a Swift concurrency pool thread → undefined behaviour, segfaults, memory corruption.
**Fix:** Mark `AppIndicatorTray` `@MainActor` (or its public methods) and dispatch through `DispatchQueue.main.async` from the actor.

### [P0] `/sessions` skips `isValidRepoKey` on `req.repoKey`
**Files:** `apple/ClawdmeterMac/AgentControl/AgentControlServer.swift:2901, 1542, 4501`
**Issue:** `handlePostSession` uses `req.repoKey` as cwd, worktree root, registry state, and OpenCode repo mapping without calling `Self.isValidRepoKey`. The sibling readonly continuation path *does* validate against traversal/control bytes/symlinks/out-of-root.
**Why it's a bug:** An authenticated paired client (compromised iPhone, stolen pairing token, attacker-paired device left briefly in front of the Mac) can `POST /sessions` with `repoKey="/tmp"` or a symlink to anywhere on the FS, and the Mac will spawn an agent rooted there. Auth + boundary failure.
**Fix:** Add `guard Self.isValidRepoKey(req.repoKey) else { return 400 }` immediately after decoding `NewSessionRequest`, before either OpenCode or tmux/worktree branch. Regression test against `/tmp`, symlink, traversal payloads.

### [P0] `/design/import-folder` accepts arbitrary `baseDir`; bridge marks it `fromTrustedPicker: true`
**Files:** `apple/ClawdmeterMac/AgentControl/AgentControlServer.swift:2766`; `tools/clawdmeter-bridge-host/index.js:163`
**Issue:** Endpoint accepts any non-empty `baseDir`; bridge mints a desktop-import token and posts `{ baseDir, fromTrustedPicker: true }` to Open Design.
**Why it's a bug:** Paired iOS device can issue `POST /design/import-folder {"baseDir":"/Users/<user>/.ssh"}` and Open Design ingests the contents because the bridge fraudulently asserts trusted-picker provenance. Data-exfil pivot.
**Fix:** Constrain `baseDir` to known repo/session roots or require a Mac-side native picker consent for each baseDir. Never set `fromTrustedPicker: true` for remote-provided paths.

### [P0] Codex→Design (Open Design) handoff non-functional end-to-end
**Files:** `tools/clawdmeter-bridge-host/index.js:74–75`
**Issue:** The daemon-IPC registration endpoint returns `{ ok: false }` unconditionally. The renderer plugin (`tools/clawdmeter-open-design-plugin/renderer.js`) silently fails on `postMessage`.
**Why it's a bug:** "Open in Code" plugin button is shipped but the round-trip never authenticates. The feature is dark with no user-visible error.
**Fix:** Complete daemon-side registration or feature-flag the plugin UI off until the round-trip lands.

### [P0] Force-unwrap on empty `applicationSupportDirectory` array
**File:** `apple/ClawdmeterShared/Sources/ClawdmeterShared/Cache/FirstPromptCache.swift:101`
**Issue:** `FileManager.default.urls(for: .applicationSupportDirectory, in: .userDomainMask).first!`
**Why it's a bug:** Sandboxed widget extensions / Catalyst configurations can return an empty array. Crash on first widget render.
**Fix:** `guard let url = fm.urls(...).first else { return nil }`.

### [P0] Force-unwrap on `TaskGroup.next()!` inside `withThrowingTaskGroup`
**File:** `apple/ClawdmeterShared/Sources/ClawdmeterShared/AgentControl/AgentControlClient.swift:1126`
**Issue:** `let first = try await group.next()!`. Returns nil when group is empty / drained.
**Fix:** `guard let first = try await group.next() else { throw … }`.

### [P0] Force-unwrap on dictionary fallback in Tahoe bindings
**File:** `apple/ClawdmeterShared/Sources/ClawdmeterShared/Tahoe/TahoeBindings.swift:61`
**Issue:** `let d = TahoeDemo.liveData[provider] ?? TahoeDemo.liveData[.claude]!`
**Fix:** Guard the fallback; show empty view on miss.

### [P0] `BrainPlanParser.latestMTime` fragile force-unwrap pattern + family of silent `try?` decodes
**File:** `apple/ClawdmeterShared/Sources/ClawdmeterShared/Brain/BrainPlanParser.swift:351` (and 152–153, 309, 315, 331, 335–336, 367–374 for the silent decodes)
**Issue:** `if latest == nil || date > latest! { … }` is a refactor-grenade; surrounding plan-cache decodes silently swallow corrupted files.
**Fix:** `if let l = latest { … } else { … }` plus logged catches.

### [P0] iOS App Transport Security disabled globally
**File:** `apple/ClawdmeteriOS/Info.plist:32–36`
**Issue:** `NSAllowsArbitraryLoads = true` with no `NSExceptionDomains`. The plist comment justifies it for Tailscale; the flag covers every other URLSession in the app.
**Why it's a bug:** Any future analytics / OAuth refresh / third-party SDK URL silently downgrades to plaintext; an on-path attacker can MITM without a cert error.
**Fix:** Replace with `NSExceptionDomains` scoped to Tailscale (`ts.net` if MagicDNS, else accept self-signed pinned cert over CGNAT).

### [P0] macOS app sandbox disabled in shipped entitlements
**File:** `apple/ClawdmeterMac/ClawdmeterMac.entitlements:9–10`
**Issue:** `com.apple.security.app-sandbox = false`. The plist comment calls it a "dev build" choice, but `dist/` shows many shipped DMGs at this entitlement.
**Why it's a bug:** Any code-execution exploit (e.g. a checksum bypass against the bundled Node from P0 above) escalates to full-user blast radius (Documents, ~/Library, browser cookies, ssh keys) instead of being sandbox-confined.
**Fix:** Split dev vs release entitlements via Xcode build configs; enable sandbox in release; ship our own OAuth flow or request a shared access group with Claude Code per the existing comment.

### [P0] GitHub Actions: third-party action pinned to mutable `@latest`
**File:** `.github/workflows/linux.yml` (`SwiftyLab/setup-swift@latest` in `shared-tests` and `linux-pkg` jobs)
**Issue:** Mutable ref; whoever controls the tag controls our CI step.
**Why it's a bug:** With no `permissions:` block (below), the default `GITHUB_TOKEN` has broad scopes. Tag re-point → CI takeover → repo write → downstream code injection.
**Fix:** Pin to a commit SHA. Refresh on intentional upgrade.

### [P0] Codex SDK relay logs full prompt bodies via OSLog `.public`
**File:** `apple/ClawdmeterMac/AgentControl/CodexSubscriptionRelay.swift:411`
**Issue:** `writeLine` logs the JSON payload (`prompt`, `workingDirectory`) with `.public`. Prompts routinely contain secrets, tokens, code, and PII.
**Why it's a bug:** Anyone with Console access (other macOS users, support tools, sysdiagnose collection) can read every user's prompt history. Privacy-as-bug.
**Fix:** Log metadata only: op, session-id, prompt length, cwd hash. Mark prompt/stdout/stderr fields `.private` (or redact).

# P1 — Important (wrong behaviour, leaks, races, broken diagnostics)

## Security / auth-boundary

### [P1] GitHub Actions workflow has no explicit `permissions:` block
**File:** `.github/workflows/linux.yml`
**Why it's a bug:** Falls back to org default (often `contents: write`). Combined with the `@latest` pin above, any CI compromise becomes a write event.
**Fix:** `permissions: contents: read` at workflow top; widen per-job only where needed.

### [P1] Pairing QR token has no length/format validation; host not constrained to Tailscale range
**File:** `apple/ClawdmeteriOS/PairingScannerView.swift:95–120`
**Why it's a bug:** A malicious printed QR (sticker over the real one, screenshot shared in chat) can redirect pairing to `attacker.example` on attacker-chosen ports. The Mac-side HKDF check is sound, but the iOS side trusts whatever's in the QR. No side-channel confirmation.
**Fix:** Pattern-check `token`/`dt` (`^[A-Za-z0-9_-]{N}$` for known N); require host ∈ Tailscale CGNAT (`100.64.0.0/10`) or `*.ts.net`; show a 4-digit confirm code on iPhone that must match Mac.

### [P1] Non-thread-safe global `umask()` mutation in Linux token writers
**Files:** `linux/Sources/ClawdmeterLinux/Storage/LinuxSecretServiceTokenProvider.swift:127`; `linux/Sources/ClawdmeterLinux/Storage/PairingTokenStore+SecretService.swift:97`
**Why it's a bug:** `umask` is a process-wide value. Hummingbird sockets / observer settings created concurrently can land with wrong permissions or fail.
**Fix:** Drop the `umask` dance; write then `setAttributes([.posixPermissions: 0o600])`, or use `open(O_CREAT, 0o600)` directly.

### [P1] Autopilot 15-minute inactivity timeout is dead code
**File:** `apple/ClawdmeterMac/AgentControl/AutopilotState.swift`
**Why it's a bug:** `recordActivity(sessionId:)` and `expiredSessions(now:)` exist but are called from nowhere. Runaway agents on idle terminals run indefinitely.
**Fix:** Wire `recordActivity` into `SessionChatStore` event ingestion; start a periodic sweep task from `AgentControlServer.start()` that disables expired autopilot sessions and emits `statusChanged`.

### [P1] Stale Anthropic token cached on Keychain delete failure
**File:** `apple/ClawdmeterShared/Sources/ClawdmeterShared/Auth/PastedAnthropicTokenProvider.swift:180–185`
**Why it's a bug:** `deleteFromKeychain()` only clears `cached` on `errSecSuccess` / `errSecItemNotFound`. Locked Keychain returns other statuses; "Sign out" appears to succeed but the singleton keeps serving the old token until restart. iCloud KC may re-sync it back.
**Fix:** Nil `cached` on every error path.

## AgentControlServer integration surfaces

### [P1] OpenCode sessions exposed but `/sessions/:id/send` has no `.opencode` branch
**Files:** `apple/ClawdmeterShared/Sources/ClawdmeterShared/AgentControl/Protocol.swift:667`; `apple/ClawdmeterMac/AgentControl/AgentControlServer.swift:1266, 2581, 2909`
**Why it's a bug:** `/sessions` will spawn an OpenCode session (no pane), then `handleSendPrompt` falls through to the tmux pane guard and 500s. Provider visibly broken end-to-end.
**Fix:** Add an `.opencode` send branch that maps Continuum UUID → OpenCode session id and dispatches via OpenCode HTTP API. Until then, hide OpenCode from session-create UI.

### [P1] OpenCode chat snapshots stay empty despite SSE `message.added`
**Files:** `apple/ClawdmeterShared/Sources/ClawdmeterShared/AgentControl/SessionFileResolver.swift:79`; `apple/ClawdmeterMac/AgentControl/OpencodeSSEAdapter.swift`; `apple/ClawdmeterMac/AgentControl/DaemonChatStoreRegistry.swift`
**Why it's a bug:** OpenCode resolves no transcript URL; SSE handler only emits a snapshot nudge but never stores/fetches the body. Snapshot endpoint returns `[]`.
**Fix:** Add an OpenCode-backed chat store (or explicit snapshot branch) that fetches OpenCode messages and writes the same snapshot counter path Claude/Codex/Gemini use.

### [P1] OpenCode crash recovery never restarts the server
**File:** `apple/ClawdmeterMac/AgentControl/OpencodeProcessManager.swift:344`
**Why it's a bug:** `handleUnexpectedExit()` clears `serveProcess`/`serverPort` but leaves `state == .running`. Next `ensureRunning()` returns early on stale state and never spawns a new `opencode serve`.
**Fix:** Set `state = .stopped`/`.starting`, clear port, then `ensureRunning()`.

### [P1] Open Design iOS forwarder collides with AgentControl WebSocket port
**Files:** `apple/ClawdmeterMac/AgentControl/AgentControlServer.swift:35, 269`; `apple/ClawdmeterMac/AgentControl/OpenDesignDaemonManager.swift:391`
**Why it's a bug:** HTTP defaults to 21731, WebSocket binds at `httpPort + 1` = 21732. `DesignPortForwarder` hardcodes 21732. Either fails to bind or layers two protocols on one port.
**Fix:** Allocate/probe a separate forwarder port outside the AgentControl fallback range; publish the chosen port.

### [P1] Open Design startup failure paths leak lock/process state
**File:** `apple/ClawdmeterMac/AgentControl/OpenDesignDaemonManager.swift:195, 263, 745`
**Why it's a bug:** Several failure paths after lock acquisition call `setFailed` without releasing the lock or terminating spawned children. Daemon health timeout leaves `daemonProcess`/`daemonPort` populated.
**Fix:** Cleanup-on-failure: terminate daemon/bridge/forwarder, clear atomics, release lock unless startup reached ready/attached.

### [P1] Plan-approval rollout-swap broken on Mac UI
**File:** `apple/ClawdmeterMac/SessionsView.swift:1148–1178`
**Why it's a bug:** `approvePlan` spawns the execution window and updates the registry, but `SessionsModel.chatStore(for:)` has a fast cache hit returning the pre-approval store that's still tailing the plan-mode JSONL. Chat freezes on the plan; only fix is app restart.
**Fix:** In `chatStore(for:)`, compare `existing.currentFileURL` against `SessionChatStore.resolveSessionFileURL(repoCwd:)` and call `switchTailedFile(to:)` when they differ.

### [P1] Plan-approval rollout-swap broken on daemon WebSocket path
**File:** `apple/ClawdmeterMac/AgentControl/DaemonChatStoreRegistry.swift:185–189`
**Why it's a bug:** `snapshotStore(for:)` file-swap check is gated on `session.kind == .chat`. `.code` sessions skip it and keep the stale resident store; iOS WS clients on `chat-subscribe` see no execution turns.
**Fix:** Drop the `.chat`-only gate; perform the file-swap check on all resident sessions using `resolveURL(session.id, session)`.

### [P1] Cold transcript loading is unbounded despite `limit`
**Files:** `apple/ClawdmeterMac/AgentControl/AgentControlServer.swift:2809`; `apple/ClawdmeterMac/AgentControl/TranscriptLoader.swift:18`; `apple/ClawdmeteriOS/iOSChatTranscriptView.swift:427`
**Why it's a bug:** `TranscriptLoader` reads the whole JSONL into memory and caps only after parse/sort. `limit=500` doesn't bound bytes/CPU/memory.
**Fix:** Streaming/tail parser with byte + message budgets; validate the path; cache only the parsed tail.

### [P1] A/B pair winner CAS does not persist the actual winner
**File:** `apple/ClawdmeterMac/AgentControl/AgentSessionRegistry.swift:304`
**Why it's a bug:** First decision stores `abPairDecidedAt` only. Later conflict responses infer the existing winner from the *new* request body, so a retry with the opposite `winnerSessionId` reports the wrong winner.
**Fix:** Persist `abPairWinnerSessionId` atomically with `abPairDecidedAt`; return stored value on 409.

### [P1] Gemini/Antigravity conversations start with synthetic prompts
**Files:** `apple/ClawdmeterMac/AgentControl/AgentControlServer.swift:1470, 3627`; `apple/ClawdmeterMac/AgentControl/LanguageServerClient.swift:213`
**Why it's a bug:** `newConversation(prompt: "(starting new chat)")` etc. invokes the agent API with that placeholder, polluting context/quota/results before the real user prompt.
**Fix:** Lazy-create on first real send; use a true empty-conversation API where available.

### [P1] Antigravity SDK mode reports active while runtime still uses disk estimates
**Files:** `apple/ClawdmeterMac/AntigravitySDKSettingsView.swift:51`; `apple/ClawdmeterMac/AgentControl/AntigravitySidecarManager.swift:172`; `apple/ClawdmeterMac/AgentControl/AgentControlServer.swift:4675`

### [P1] Bridge-host parent-PID watchdog uses Linux-only `/proc`
**File:** `tools/clawdmeter-bridge-host/index.js:301–312`
**Why it's a bug:** On macOS (primary target) `/proc` doesn't exist; the check is a no-op and the bridge happily orphans when the Mac app crashes, keeping the loopback port.
**Fix:** `process.ppid` + `kill(ppid, 0)`; exit on `ESRCH`.

### [P1] Bridge-host HMAC secret derived from observable PIDs
**File:** `tools/clawdmeter-bridge-host/index.js:129–141`
**Why it's a bug:** `pids.slice(0,16).join('').padEnd(32,'0')` as HMAC key. PIDs are public to any local process via `ps`. Trivial token forgery.
**Fix:** `crypto.randomBytes(32)` persisted at chmod 0600.

### [P1] Bridge-host HTTP server has no timeouts, no connection caps, no body size limit
**File:** `tools/clawdmeter-bridge-host/index.js:10–20`
**Fix:** `setTimeout(30_000)`, `headersTimeout = 10_000`, `maxConnections = 16`, 1 MB body cap.

### [P1] codex-sdk: `workingDirectory`/`prompt` not validated before SDK
**File:** `tools/clawdmeter-codex-sdk/main.mjs:114–117`
**Fix:** `path.resolve` then assert under `$HOME` / project root; cap prompt length.

### [P1] codex-sdk: `readline` with `crlfDelay: Infinity` allows unbounded line buffering
**File:** `tools/clawdmeter-codex-sdk/main.mjs:215`
**Fix:** Per-line cap (e.g. 1 MB); close stdin with error on overflow.

### [P1] codex-sdk: `firstLine` flag race for header handler
**File:** `tools/clawdmeter-codex-sdk/main.mjs:216, 231–232, 269–277`
**Fix:** `rl.once('line', headerHandler)`; attach payload handler from inside.

### [P1] open-design plugin: `postMessage` lacks nonce/auth and trusts unvalidated globals
**File:** `tools/clawdmeter-open-design-plugin/renderer.js:48–58`
**Why it's a bug:** Any other plugin or compromised page script can spoof a Code-handoff. `window.__OD_ACTIVE_PROJECT__` is trusted as-is.

**Files:** `apple/ClawdmeterMac/AgentControl/PseudoTerminal.swift:26`; `apple/ClawdmeterMac/AgentControl/TmuxControlClient.swift:209`
