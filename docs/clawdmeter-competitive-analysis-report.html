01 Executive Summary

Continuum occupies a highly unique, developer-centric niche: it is a native, local-first control plane, telemetry ledger, and multi-device companion cockpit for heterogeneous coding agents (Claude Code, Codex, Gemini/Antigravity, and OpenCode).

The core product wedge is visceral and immediate: "Run heavy agents locally on your Mac; monitor, approve, and steer them from your iPhone or Apple Watch while you walk away."

However, to become a dominant, high-leverage developer platform, Continuum must be strategically positioned and engineered against two dominant, highly capitalized product paradigms:

CR
Cursor

An AI-first VS Code fork that owns the editor loop. It excels at low-latency autocomplete, active file buffer modifications (Composer), and automated codebase vector indexing.

CO
Conductor (Melty Labs)

A desktop-only workbench designed specifically to coordinate parallel agents. It automates git worktree sandboxing and represents agents as virtual developers on a board.

03 Codebase & Architecture Assessment

Continuum is powered by a solid macOS daemon. The code shows highly advanced, low-level coordination primitives, but highlights specific areas of structural complexity and deferred implementation.

SYSTEM TOPOLOGY BLUEPRINT
1. Control & runtime supervision

The daemon uses direct PTY hosts and provider harnesses to orchestrate local agent sessions in the background. It reads local transcripts and wires events dynamically.

2. Worktree Provisioning

WorktreeManager.swift provisions isolated git worktrees, writes ownership manifests, and implements a multi-gate garbage-collection schema to avoid deleting user modifications.

3. Local ccusage Cost Ledger

UsageHistoryLoader.swift parses local JSONLs and SQLite Trajectories in parallel task groups, pricing tokens using an embedded LiteLLM model map snapshot with 0% data exfiltration.

04 Continuum's Unfair Advantages

We shouldn't compete with Cursor on editor design, or with Conductor on desktop-only fleet layout. Instead, Continuum must double down on its highly defensible, native Apple-device ecosystem.

Glanceable Control

Live Activities & Dynamic Island display token burns, compilation steps, and execution states. High-priority Watch complications let you steer worktrees with a crown-press or wrist-tap.

Broadcaster Consensus

Continuum's broadcast chat comparative engine lets you issue a single task to Claude, Codex, and Gemini side-by-side, benchmark outputs, and promote the winner to a live session.

Cost & Quota Ledger

Fully offline token tracking that monitors weekly limits. Before firing off a large run, preflight diagnostics estimate cost projections, pricing the swarm accurately by provider and repository.

05 Opportunities to Defeat Conductor

Conductor treats AI agents as parallel virtual junior developers on a desktop board. Continuum can dismantle this wedge by taking this exact orchestration concept and mirroring it to the Apple device ecosystem.

OPPORTUNITY 01

The Mobile Fleet Kanban Board

Map Continuum's worktrees into a multi-device Kanban dashboard: Backlog ➔ In Progress ➔ Review ➔ Done. Represent running agent processes as distinct visual developer cards (with avatars and city-name branch labels). Mirror the technical lead's steer board directly to iOS.

Status: Proposed High Value
OPPORTUNITY 02

Post-Provisioning Script Execution

Closes the top functional gap in Continuum's worktree manager. Read conductor.json or a custom workspace script file to automatically trigger npm install, build targets, or setup environments inside the sandboxed folder before runtime spawning starts.

Status: Planned Essential Parity
OPPORTUNITY 03

Consensus Synthesis Diffs

When broadcasting prompts to multiple agents (e.g. Claude on branch A, Codex on branch B), display hunktrees side-by-side. Build a consensus merger tool, allowing developers to cherry-pick files and hunks from different runs to combine into a final merged output.

Status: Conceptual UX Delighter
OPPORTUNITY 04

Watch Crown Actions & Complications

Expand watchOS complications to display aggregations of the agent swarm ("3 running, 1 plan waiting"). Tapping the complication launches a scrollable list. Approve the plan with a simple double-tap or steer it directly from your wrist.

Status: Proposed Wearable Wedge

06 Opportunities to Complement Cursor

Do not try to build a better text editor than Cursor. Instead, position Continuum as the ultimate background companion cockpit that hooks into the editor writing loop.

05

"Send to Continuum" Shim

Create a lightweight VS Code extension. Highlight code, press a hotkey, and offload the task to Continuum. The extension spawns background worktree runs so you never freeze your active editor layout.

06

Embeddings-in-a-Box (RAG)

Integrate a lightweight, sandboxed vector engine (e.g. SQLite-VSS) into the Mac daemon. Build a background semantic indexer of the repo, enabling mobile users to command agents using rich context commands like @Codebase.

07

Swarm Loop CPU Health Monitor

Monitor spawned agent processes for CPU spikes and endless loops. If an agent gets caught in a failing test cycle and overheats your Mac, send an alert to iPhone with a one-tap "Kill & Rollback" trigger.

07 Concrete Tactical Action Plan

PHASE 1: HARDEN THE HOOKS

Deliver Zero-Latency Push & Provisioning Scripts

Deploy the client implementations for RelayClient.swift (E3/E4) and APNSGatewayClient.swift (E6). Resolve the XChaCha20-Poly1305 cross-implementation gap in Swift by adding libsodium-swift or implementing the subkey prelude natively. Add post-provisioning scripts to worktrees to run setup steps before the agent spawns.

PHASE 2: ORCHESTRATE & EMBED

Build the Fleet Board & Sandboxed RAG

Implement the Kanban Fleet Dashboard in the Mac UI and mirror the status groupings to the iPhone. Wire a sandboxed local vector indexer (SQLite-VSS) into the repository indexing actor, adding support for the @Codebase search commands in the composer.

PHASE 3: EDITOR HOOKS

Create the Cursor Extension & Consensus Merger

Publish the "Send to Continuum" TypeScript shim extension on the VS Code Marketplace. Implement the Hunk Synthesis editor in Continuum, giving developers a dedicated comparative panel to reconcile branches and commits from multiple agent rollouts.
